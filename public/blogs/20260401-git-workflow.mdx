很多人觉得 Git 是团队协作才用的工具，自己一个人写代码没必要。错！个人项目用 Git 至少有三个好处。

第一，版本回退。改了一堆代码结果把功能改坏了？`git reset` 一键回到上一个稳定版本，比 Ctrl+Z 强大太多。第二，分支开发。想在项目里试验一个新功能又怕搞砸主分支？开个 feature 分支，做成了合并，做废了删掉，完全不影响主干。

第三，代码备份。GitHub 免费私有仓库，代码推上去就多一个异地备份，电脑坏了也不怕。

我每天的工作流很简单：早上 `git pull` 同步，写代码时频繁 `git add -p` 小块提交，傍晚 `git push` 推到远程。提交信息用中文写清楚改了什么，一个月后回头看还能理解当时的思路。
