传统 Markdown 只能写静态内容，而 **MDX 可以直接在文章里使用 React 组件**，让博客从"文字页面"升级为"互动体验"。下面用 11 个组件演示 MDX 的独特能力。

## 1. 提示框 Callout

写技术文章经常需要插入提示、警告或小技巧，用 Callout 组件可以醒目地展示：

</Callout>

<Callout type="tip" title="小技巧">

</Callout>

## 3. 折叠面板 Accordion

适合放 FAQ 常见问题，用户按需展开，不浪费页面空间：

<Accordion title="MDX 和普通 Markdown 有什么区别？">
MDX 是 Markdown 的超集，完全兼容 Markdown 语法，同时允许在文中直接使用 JSX 组件。这意味着你可以在文章里嵌入任何 React 组件，比如图表、表单、动画等。
</Accordion>

<Accordion title="博客用什么工具渲染 MDX？">
本项目使用 <Kbd>next-mdx-remote</Kbd>，它支持在 Next.js 服务端组件中安全地渲染 MDX，是目前最主流的方案之一。
</Accordion>

<Accordion title="如何在 MDX 中自定义组件？">
在 `MDXRemote` 的 `components` 属性中注册即可。组件可以是服务端组件，也可以是客户端组件（加 `"use client"`）。
</Accordion>

## 5. 代码块复制按钮

鼠标悬停在代码块上，右上角出现复制按钮，一键复制整段代码：

```tsx
// 鼠标悬停代码块试试右上角
interface BlogPost {
  title: string;
  date: string;
  content: string;
}

function parseMdx(file: string): BlogPost {
  const { data, content } = matter(file);
  return { title: data.title, date: data.date, content };
}
```
